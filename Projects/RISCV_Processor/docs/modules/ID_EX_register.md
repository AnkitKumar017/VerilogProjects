# ID/EX Pipeline Register

## 1. Purpose

The ID/EX Pipeline Register stores all outputs of the Instruction Decode (ID) stage and passes them to the Execute (EX) stage on the next clock cycle.

---

## 2. Location

Source File

src/pipeline/id_ex.v

Testbench

tb/id_ex_tb.v

---

## 3. Inputs

| Signal          |   Width | Description                           |
| --------------- | ------: | ------------------------------------- |
| clk             |       1 | System clock                          |
| rst             |       1 | Active-high reset                     |
| pc              |      32 | Current Program Counter               |
| read_data1      |      32 | Register File output 1                |
| read_data2      |      32 | Register File output 2                |
| imm_ext         |      32 | Sign-extended immediate               |
| rs1             |       5 | Source Register 1                     |
| rs2             |       5 | Source Register 2                     |
| rd              |       5 | Destination Register                  |
| Control Signals | Various | Signals generated by the Control Unit |

---

## 4. Outputs

Same as inputs, delayed by one clock cycle.

---

## 5. Internal Operation

Captures all Decode stage outputs on every rising edge of the clock.

---

## 6. Timing

Sequential Logic

Triggered on:

posedge clk

## 8. Verification Strategy

## 9. Design Notes

This register isolates the Decode and Execute stages and enables pipelined execution.
